This guide walks you through presenting a basic popover using both SwiftUI and UIKit. You will have something on screen by the end of step three. For deeper customization — like positioning, dismissal mode, and animations — see the [position reference](/customization/position) and the [full SwiftUI guide](/presenting/swiftui).

<Steps>
  <Step title="Install Popovers">
    Follow the [installation guide](/installation) to add Popovers to your Xcode project via Swift Package Manager or CocoaPods.
  </Step>

  <Step title="Import the library">
    Add the import at the top of your Swift file:

    ```swift
    import SwiftUI
    import Popovers
    ```
  </Step>

  <Step title="Present your first popover">
    Choose the approach that matches your project:

    <Tabs>
      <Tab title="SwiftUI">
        Attach the `.popover(present:)` modifier to any view. Pass a `@State` `Bool` binding — set it to `true` to show the popover and `false` to dismiss it.

        ```swift
        import SwiftUI
        import Popovers

        struct ContentView: View {
            @State var present = false

            var body: some View {
                Button("Present popover!") {
                    present = true
                }
                .popover(present: $present) {
                    Text("Hi, I'm a popover.")
                        .padding()
                        .foregroundColor(.white)
                        .background(.blue)
                        .cornerRadius(16)
                }
            }
        }
        ```

        By default, the popover anchors itself to the view the modifier is attached to — no extra configuration needed.
      </Tab>
      <Tab title="UIKit">
        Create a `Popover` struct with a SwiftUI `View` as its content, set the `sourceFrame` so the popover knows where to anchor, then call `present(_:)` on your view controller.

        ```swift
        import SwiftUI
        import Popovers

        class ViewController: UIViewController {
            @IBOutlet weak var button: UIButton!
            @IBAction func buttonPressed(_ sender: Any) {
                var popover = Popover { PopoverView() }
                popover.attributes.sourceFrame = { [weak button] in
                    button.windowFrame()
                }

                present(popover)
            }
        }

        struct PopoverView: View {
            var body: some View {
                Text("Hi, I'm a popover.")
                    .padding()
                    .foregroundColor(.white)
                    .background(.blue)
                    .cornerRadius(16)
            }
        }
        ```

        <Note>
          Always use `[weak button]` in the `sourceFrame` closure to avoid a retain cycle.
        </Note>
      </Tab>
    </Tabs>
  </Step>
</Steps>

## State management in SwiftUI

If your popover's content depends on state that changes after the popover is presented, move the content into its own struct and pass state down as a `Binding`. Passing a plain value (without `$`) means the popover view will not re-render when the value changes.

```swift
struct ContentView: View {
    @State var present = false
    @State var string = "Hello, I'm a popover."

    var body: some View {
        Button("Present popover!") { present = true }
        .popover(present: $present) {
            PopoverView(string: $string) // Pass a Binding ($string), not a plain value
        }
    }
}

struct PopoverView: View {
    @Binding var string: String

    var body: some View {
        Button(string) { string = "The string changed." }
            .background(.mint)
            .cornerRadius(16)
    }
}
```

<Note>
  Always put the popover's content in a separate struct and pass mutable state as a `@Binding`. Passing a non-binding variable directly will cause the popover to miss state updates.
</Note>
